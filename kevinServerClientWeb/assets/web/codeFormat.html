代码格式说明

在 大型的/多人合作 的项目中, 有一个统一的代码格式是必不可少的, 此文章列出了所有在本项目中的代码格式

    请使用正确的元素标签

  • 使用自带的元素标签来导入文件


    对于CSS(叠层样式表)文件, 请使用<style>来导入

    具体方法为:

    <style>
    @import url("../css/*文件名.css");
    </style>

    请注意: 在CSS中请不要在每个类之间添加空格, 这是不必要的, 垂直格式已经足够


  • 而JS(JavaScript)文件, 请使用<script>来导入

    具体方法为:

    <script src="../js/*文件名.js"></script>

    • 请不要使用 <link> 标签, 因为此标签只适用于特殊文件的导入且只能用于特殊的用途 (比如 网页/标签页 图标的导入和加载)

    还有别弄 faction.ico ! 这玩意就是个废


    请使用正确的格式

  • 请确保使用英文标点符号以及空格


    英文的标点符号更通用, 而且某些中文的标点符号容易和英文的混淆, 导致字体显示出错

    这是一个例子,中文的标点符号会让文案阅读起来更困难（更何况大家代码写多了应该都看不惯吧？？？）.//《-中文的斜杠哈

    最关键的是, VSC有时无法为中文符号加上高亮↓, 这导致了更正的工作量加大...


  • 请在不同的符号之间添加空格


    不加空格, 会让文章显现的很紧凑, 难以阅读 (堆成一堆了).

    //这又是一个例子,显示了不加空格对于代码的难以阅读性
    window.addEventListener("mousedown",function(event){
    const number=99;
    for(i=0;i<number;i++){
    console.log("testingLOL");
    }
    });

    但也不要添加过多的空格

    添加过多的缩进也会梳离各个语句之间的关系

    例子:

    let a = 90 ;
    a += 90 ;
    console . log ( a + 'null' ) ;

  • 主要规则如下:

    • 在显示多个不同的 选项/选择 时, 请在之间使用 / 并且在前后添加空格
    • 在显示单或多个符号时, 请将其填入 <code>, 并且左右添加空格
    • 在断句时, 添加标点符号, 如逗号 , 和句号 ., 而不是空格
    • 但是在一段句子结束之后, 你可以不使用句号 .
    • 在使用括号 () 时, 也要在左右添加空格
    • 在高亮某个文本时 (即 <strong> 元素) , 如果旁边有标点符号, 请添加进去
    • 注: 请不要使用 <b> 元素, 难维护
    • 但是在划去文本则不用添加标点符号 (即 <del> 元素)
    • 不要使用水平线 (即 <hr> )

    • 如果多个不同的标点符号在一起, 可以不使用空格
    • 在左花括号 {, 前请添加空格
    • 必要的时候, 你可以使用转义符号

    • 请正确使用注释

  • 代码本身就已经说明了它的作用, 在使用注释前, 请确保你要说明的内容是较难已理解的我不是傻子

    比如多个且复杂的 类/函数/定义 等

    或者是在容易出错或误解的地方添加注释来提醒他人


    注意: 在注释内容左右要加空格

    如下:

    // 这是一行注释
    /* 这是一行注释 */
    <!-- 这是一行注释 -->

注: 对于对某些语言不熟悉的贡献者, 建议找专业的人来编辑吧...


*: 只为文件名, 不是后缀
